## Описание проекта

Лабораторная работа включает два интерактивных веб-приложения: калькулятор и крестики-нолики, разработанных с использованием HTML, CSS и JavaScript.

## Калькулятор

### Функциональность

- Поддерживает операции: сложение, вычитание, умножение, деление
- Поддерживает десятичные числа
- Кнопка очистки (C) сбрасывает выражение
- Кнопка (=) вычисляет результат

### Ограничения

- Нельзя вводить два оператора подряд
- Нельзя вводить несколько точек в одном числе
- Выражение не может начинаться с оператора (кроме -)

### Обработка ошибок

- При делении на ноль отображается "Ошибка"
- При некорректном выражении отображается "Ошибка"
- После ошибки калькулятор сбрасывается при следующем нажатии

### Класс Calculator

- `constructor()` - инициализация полей: display, expression, hasError
- `init()` - навешивает обработчики на кнопки
- `handleButtonClick(value)` - распределяет действия по нажатию
- `addToExpression(value)` - добавляет символ с валидацией
- `calculate()` - вычисляет выражение
- `safeEvaluate(expr)` - изолируетвызов eval с проверкой
- `isValidExpression(expr)` - проверяет корректность выражения
- `clear()` - сбрасывает состояние
- `updateDisplay()` - обновляет поле вывода
- `showError(message)` - переводит в состояние ошибки

### Схема взаимодействия
index.html > main.js > Calculator.init()
button.click > handleButtonClick()
addToExpression() / calculate() / clear()
updateDisplay()

## Логика работы

### Последовательность при нажатии кнопки

1. Пользователь нажимает кнопку
2. `handleButtonClick()` получает значение
3. Если была ошибка - сбрасывает состояние
4. Распределяет: C > clear, = > calculate, иначе > addToExpression
5. `updateDisplay()` обновляет экран

### Валидация ввода

- Два оператора подряд - второй заменяет первый
- Точка уже есть в числе - игнорируется
- Оператор в начале (кроме -) - игнорируется

### Вычисление

1. Проверяется валидность выражения через `isValidExpression()`
2. Вызывается `eval()` внутри `safeEvaluate()`
3. Проверяется конечность результата через `isFinite()`
4. При ошибке вызывается `showError()`
